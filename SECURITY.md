# Security Policy

## Reporting

Сообщайте об уязвимостях через GitHub Private Vulnerability Reporting этого
репозитория. Не публикуйте issue с эксплуатационными деталями до выпуска
исправления. Не прикладывайте реальные секреты, персональные данные 1С или дампы
production-баз.

В отчёте укажите затронутый commit/image digest, сценарий воспроизведения,
влияние и безопасный proof of concept. Владелец подтверждает получение и
согласовывает сроки исправления и coordinated disclosure.

## Supported Version

Исправления безопасности выпускаются для актуального состояния ветки `main`.
Развёртывания должны использовать immutable SHA image tag или digest, а не
плавающий `latest`.
